# Requisitos de Processo — Sistema Urbanly

## RNP001 — Processo de Desenvolvimento

O sistema deve ser desenvolvido seguindo uma **arquitetura modular**, separando os seguintes componentes:

* frontend
* backend
* banco de dados

Essa separação deve facilitar:

* manutenção do sistema
* testes
* futuras expansões.

---

## RNP002 — Metodologia de Desenvolvimento

O desenvolvimento do sistema deve seguir uma **metodologia incremental**, permitindo a entrega de funcionalidades em etapas.

O desenvolvimento deve iniciar com um **MVP (Minimum Viable Product)** e evoluir gradualmente com novas funcionalidades.

---

## RNP003 — Controle de Versão

O código-fonte do sistema deve ser gerenciado utilizando um **sistema de controle de versão**, como Git.

Esse controle deve permitir:

* rastreamento de alterações
* colaboração entre desenvolvedores
* recuperação de versões anteriores.

---

## RNP004 — Padronização de Código

O sistema deve seguir **padrões de codificação e boas práticas de programação**, garantindo:

* legibilidade
* organização
* consistência do código.

---

## RNP005 — Documentação do Sistema

O projeto deve possuir documentação técnica contendo:

* descrição da arquitetura
* requisitos do sistema
* estrutura do banco de dados
* principais funcionalidades

facilitando manutenção e evolução do sistema.

---

## RNP006 — Testes do Sistema

O sistema deve passar por **testes funcionais**, verificando se as funcionalidades implementadas estão operando corretamente conforme os requisitos definidos.
