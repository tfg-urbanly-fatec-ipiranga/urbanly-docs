# Requisitos Não Funcionais — Sistema Urbanly

## RNF001 — Usabilidade

O sistema deve possuir uma interface simples, intuitiva e de fácil navegação.

O usuário deve conseguir:

* buscar estabelecimentos
* visualizar detalhes
* realizar avaliações
* navegar pelo aplicativo

sem necessidade de treinamento prévio.

---

## RNF002 — Apresentação da Interface Gráfica

A interface do aplicativo deve ser:

* responsiva
* adaptada para dispositivos móveis

Compatível com:

* Android
* iOS

A interface deve utilizar:

* tipografia legível
* ícones intuitivos
* cores padronizadas
* elementos visuais consistentes

garantindo boa experiência de uso.

---

## RNF003 — Segurança e Privacidade

O sistema deve proteger dados dos usuários por meio de mecanismos de segurança e implementar controle de acesso para diferenciar funcionalidades disponíveis para usuários autenticados e não autenticados.

Incluindo:

* criptografia de dados sensíveis
* armazenamento seguro de credenciais
* autenticação segura

O sistema deve estar em conformidade com a **Lei Geral de Proteção de Dados (LGPD)**.

---

## RNF004 — Compatibilidade

O aplicativo deve ser compatível com os sistemas operacionais:

* Android
* iOS

Devendo contemplar as **versões mais utilizadas no mercado no momento do desenvolvimento do MVP**.

---

## RNF005 — Escalabilidade

O sistema deve suportar crescimento da base de usuários e estabelecimentos cadastrados.

A arquitetura deve permitir **escalabilidade horizontal**, possibilitando:

* adição de servidores
* uso de infraestrutura em nuvem
* balanceamento de carga

sem comprometer o desempenho do sistema.

---

## RNF006 — Manutenibilidade

O sistema deve ser desenvolvido com código:

* estruturado
* padronizado
* documentado

facilitando:

* manutenção corretiva
* manutenção evolutiva
* integração futura de novas funcionalidades, como **Inteligência Artificial**.
